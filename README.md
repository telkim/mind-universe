# Mind Universe

이 프로젝트는 마크다운 글을 디지털 가든 형태로 찾아보기 쉽게 보여주는 시스템입니다.

## 🚀 로컬 개발 환경 기동

프로젝트 가동을 위해 다음 명령어를 사용합니다.

```bash
# 종속성 설치
npm install

# 개발 서버 실행
npm run dev
```

개발 서버는 일반적으로 `http://127.0.0.1:5173`에서 실행됩니다.

## 📝 마크다운 작성 규칙

마크다운 파일은 `public/` 디렉토리 내의 데이터 JSON 파일들을 기반으로 합니다. 직접적인 마크다운 편집보다는 데이터 구조에 맞춰 JSON을 업데이트하는 방식이 권장됩니다.

- **데이터 소스**: `public/data.json`, `public/ft-2026Q2.json` 등
- **구조**:
  - `postId`: 고유 식별자
  - `date`: 작성일
  - `theses`: 주제 (카테고리)
  - `title`: 제목
  - `summary`: 요약
  - `link`: 원문 링크 (없을 경우 네이버 블로그 링크로 대체됨)

## 🔄 매핑 데이터 변환 프로세스

데이터가 Graph View에 반영되는 단계입니다.

1. **데이터 업데이트**: `public/` 내의 `.json` 파일들을 수정/추가합니다.
2. **검색 데이터 빌드**: 시스템이 `search-summary.json`을 사용하여 그래프 노드와 검색 인덱스를 구성합니다.
3. **Graph View 반영**: React 컴포넌트(`src/main.jsx`)가 JSON 데이터를 읽어 D3.js 기반 로직(Canvas)으로 그래프 노드와 연결선(`line`)을 생성합니다.
4. **빌드**: `npm run build` 명령어를 통해 `dist/` 디렉토리에 정적 파일이 생성됩니다.

## 🛠️ 트러블슈팅

그래프 노드가 보이지 않거나 링크가 깨질 경우 다음을 확인하세요.

1. **데이터 경로 확인**: `public/` 내의 JSON 파일이 올바른 포맷(배열 또는 객체)인지 확인하세요.
2. **JSON 파싱 오류**: `search-summary.json` 혹은 기타 JSON 파일에 문법 오류가 없는지 확인하세요 (JSON Validator 활용).
3. **캐시 초기화**: 개발 서버 실행 후 변경 사항이 즉시 보이지 않으면 브라우저 캐시를 삭제하거나 서버를 재시작하세요.
4. **환경 변수**: `import.meta.env.BASE_URL` 기반의 `assetPath()` 함수를 사용하므로, 배포 환경에서 경로가 올바르게 설정되었는지 확인하세요 (`package.json`의 `build` 스크립트 참고).
