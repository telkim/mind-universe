
## 코딩 철학 및 Superpowers 하네스 루프 (Testing & Verification)
* **No-Scroll 뷰포트 고정**: 전체 화면은 브라우저 뷰포트(100vh)에 고정되며 세로 스크롤바가 생기지 않아야 한다. 그래프 캔버스와 우측 목록 패널이 독립적으로 레이아웃을 유지하는지 개발 전 CSS 구조를 최우선 검증하라.
* **Quartz 컴포넌트 간 상태 동기화**: 좌측 그래프에서 8개 대주제 또는 세부 테제 노드를 클릭했을 때, 우측 목록 패널 컴포넌트에 해당 테제의 글 목록이 즉각 필터링되어 동기화되는지 데이터 흐름(State/Event) 시뮬레이션을 수행하라.
* **로컬 저장소 방어적 코딩**: '나중에 읽기(⏰)', '스크랩(🔖)' 기능 구현 시, Quartz의 정적 빌드(SSG) 과정에서 `window / localStorage` 객체 접근 시 발생할 수 있는 서버 사이드 빌드 에러 및 런타임 JSON 파싱 오류를 사전에 철저히 차단하라.

## 구현 및 출력 규칙 (Implementation Rules)

* **코드 생략 및 말줄임표 (`// ...`) 절대 금지**: "기존 Quartz 코드와 동일", "중략", "D3 로직 생략" 등의 핑계로 코드를 일부만 출력하는 행위를 엄격히 금지한다. Quartz 레이아웃 컴포넌트(`Component.tsx`)부터 테제 필터 셋업, 푸터 컨트롤러(확대/축소, 글 수 카운터), 상단 통합 검색창까지 사용자가 바로 복사해서 파일에 덮어쓸 수 있는 완전한 전체 코드(Full Code)만 제공한다.
* **수정 이력 명시**: 기존 Quartz 컴포넌트나 레이아웃 로직을 수정하여 재출력할 때는, 코드 최상단 주석에 `[수정 원인]`과 `[해결 방안]`을 각각 단 한 줄씩 명확히 요약한 후 전체 코드를 전개한다.

#### 푸터 영역 (Footer & Controls)
* 시스템에 등록된 총 글의 개수를 보여주는 카운터 명시.
* **그래프 컨트롤러 최적화**: 확대(`+`), 축소(`-`), 화면 맞춤(`⤢`), 링크 고정(`🔗`) 버튼은 D3.js 줌 로직을 처음부터 새로 구현하지 말고, **Quartz 내장 그래프 컴포넌트의 API 및 상태 제어 함수를 바인딩하여 최소한의 정적 코드로 알아서 내장 기능이 작동하도록 구현**할 것 (토큰 낭비 방지).

# Search Index Data Contract & Principles

이 프로젝트의 블로그 검색은 외부 원문을 실시간 크롤링하지 않으며, 지정된 정적 JSON 검색 인덱스를 기준으로 전적으로 프론트엔드 내에서 동작한다.

## 1. 파일 구성 및 데이터 규격

### 1) `search-summary.json`
초기 로딩과 기본 검색에 사용하는 경량 메타데이터 인덱스 파일이다. 첫 화면 진입 시 반드시 가장 먼저 로드해야 한다.
```json
{
  "v": 1,
  "posts": [
    [
      "224236198290",
      "2026-04-01",
      "직업",
      "딴짓이 진짜 커리어가 되는 사람들의 비밀",
      "요즘 유튜브를 열면 \"사이드 프로젝트를 시작하라\"는 영상이 쏟아진다."
    ]
  ]
}

```

* **배열 필드 인덱스 매핑 규칙:**
* `[0]` `postId`: 네이버 블로그 logNo 또는 내부 게시글 고유 ID (String)
* `[1]` `date`: 작성일 (`YYYY-MM-DD`)
* `[2]` `theses`: 카테고리명 (테제 문자열을 슬래쉬, 괄호문자, 콤마로 구분)
* `[3]` `title`: 게시글 제목
* `[4]` `summary`: 본문 앞부분 또는 검색 결과용 요약문



### 2) `ft-manifest.json`

전체 본문(Full Text) 인덱스 파일 목록을 관리하는 메니페스트 파일이다. 사용자가 명시적으로 '본문 검색'을 요청하기 전에는 절대 미리 로드하지 않는다.

```json
{
  "v": 1,
  "files": [
    "ft-2026Q2.json",
    "ft-2026Q1.json",
    "ft-2025Q4.json"
  ]
}

```

### 3) `ft-2026Q2.json`

분기별 전체 본문 검색용 인덱스 파일이다. 데이터 중복을 최소화하기 위해 제목, 날짜, 카테고리는 포함하지 않으며 오직 매칭용 ID와 본문 텍스트만 가진다.

```json
{
  "v": 1,
  "posts": [
    [
      "224236198290",
      "요즘 유튜브를 열면 \"사이드 프로젝트를 시작하라\"는 영상이 쏟아진다. ..."
    ]
  ]
}

```

* **배열 필드 인덱스 매핑 규칙:**
* `[0]` `postId`: `search-summary.json`의 `postId`와 조인(Join)하기 위한 키
* `[1]` `body`: 게시글 전체 본문을 단일 텍스트로 평탄화(Flatten)한 값



### postId 값에 의한 링크 주소 호출
postId 에 의한 링크는 다음의 base url 을 사용하여 링크 정보를 생성한다.

* base url : https://blog.naver.com/bambooinvesting/
* 링크 형식 : <base url> + <postId>


### [데이터 분류 체계 및 옵시디언 마크다운 구조]
* **글목록 클릭 인터랙션**: 글 목록에서 특정 글을 클릭하면, "https://blog.naver.com/n-mind/" 에 blogId 값을 결합한 링크로 연결한다. 링크는 새 창 이동(`_blank`) 해야 한다.

* **마크다운 Frontmatter 포맷 규격**:
```markdown
---
postId: "224236198290"
title: "미국, 조선업판 IRA"
date: 2026-04-28
mainTheme: "기계·로봇·방산·조선·우주"
theses: "기계/로봇/방산/조선/우주"
---
본문 내용 요약이 여기에 들어갑니다.
```
