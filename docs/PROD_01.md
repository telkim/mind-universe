## 대주제 - 테제 연결 및 치환 구현 지시사항
대주제는 사업,직업,생활,국제정세,사회문화,철학,부동산,투자 8가지야.
상단 검색영역 아래에 대주제를 클릭할 수 있는 버튼을 나열해.

1. 정적 노드 구조(메뉴판)의 사전 정의
시스템은 대주제와 하위 세부 테제 간의 관계를 런타임에 동적으로 유추하지 않습니다. 개발자는 UI 좌측 및 네트워크 그래프를 구성할 8대 대주제와 그에 종속된 세부 테제 트리 구조를 자바스크립트 내에 정적(Static) 데이터로 미리 선언해 두어야 합니다.

2. 원문 카테고리를 통한 테제 노드 런타임 바인딩
search-summary.json을 로드할 때, 각 포스트의 원문 카테고리 필드(post[2]) 스트링을 읽어 사전에 정의된 대주제/테제 키워드와 비교하여 글을 알맞은 노드에 주입(Push)합니다.

슬래시 분절 매칭: post[2]가 "SW/플랫폼/게임/통신/AI"와 같이 슬래시로 연결되어 있다면, 이를 split('/')으로 쪼갠 뒤 사전에 정의된 [AI·소프트웨어·플랫폼] 대주제 내의 [AI] 또는 [플랫폼] 테제 노드에 해당 포스트 ID를 연결합니다.

3. 텍스트 치환 및 그룹화 규칙 (Alias Mapping)
search-summary.json의 원문 카테고리 명칭이 UI 상의 세부 테제 명칭과 일치하지 않는 경우, 런타임에 이를 정제된 테제 노드로 연결해 주는 치환(Alias) 딕셔너리를 거쳐야 합니다.

치환 맵 구현 규격 예시:

JavaScript
const CATEGORY_ALIAS = {
  // 원문 카테고리 핵심어 : [연결할 대주제, 연결할 세부 테제]
  "투자단상": ["투자 철학·단상", "🔥 시장을 읽는 법"],
  "사업/삶": ["사업·삶·커리어", "🔥 AI 시대의 일·커리어"],
  "매크로": ["매크로·국제정세", "🔥 트럼프·미국 정치"]
};
매칭 로직:

포스트의 post[2] 문자열에 CATEGORY_ALIAS의 Key(예: "투자단상")가 포함되어 있는지 검사합니다.

매칭이 확인되면, 해당 포스트를 고정 정의된 대주제 [투자 철학·단상] 하위의 [🔥 시장을 읽는 법] 테제 노드의 자식 배열(posts 리스트)에 동적으로 할당합니다.

최종적으로 각 테제 노드에 쌓인 포스트 배열의 길이(posts.length)를 계산하여 화면의 글 개수 카운트(예: 104, 38)를 실시간으로 렌더링합니다.



## 런타임 노드 생성 메커니즘
지침: 본 시스템은 대주제 및 테제(소주제) 데이터를 정적으로 들고 있지 않으며, 데이터 파일 내에 별도의 태그 필드도 요구하지 않습니다.

구현 요구사항: search-summary.json을 로드할 때 각 포스트의 카테고리 필드(post[2])를 읽어 슬래시(/) 기호로 분절하는 파싱 파이프라인을 구축하십시오. 분절된 각 토큰을 기반으로 네트워크 그래프의 노드를 런타임에 동적으로 생성하고, 동일한 토큰을 공유하는 포스트 ID들을 해당 노드에 매핑하여 계층 구조와 글 개수 카운팅을 자동으로 빌드하도록 엔진을 설계해야 합니다.

아래의 부연 설명을 참고 합니다.

1. 텍스트 분절을 통한 동적 트리 빌드 (Split & Map)
코드는 별도의 딕셔너리 없이, post[2]에 들어있는 원문 카테고리 문자열(예: SW/플랫폼/게임/통신/AI)을 읽자마자 슬래시(/) 기호를 기준으로 무조건 분절(split('/'))합니다.

동작 방식:

"SW/플랫폼/게임/통신/AI"를 분절하여 ['SW', '플랫폼', '게임', '통신', 'AI']라는 5개의 독립된 단어 배열을 만듭니다.

이 분절된 단어들을 순회하면서, 메모리 상에 존재하는 네트워크 그래프 데이터셋(nodes 배열)에 해당 이름의 노드가 이미 등록되어 있는지 확인합니다.

존재하지 않는 단어라면 그 즉시 런타임에 새로운 테제 노드(예: name: "AI")를 동적으로 생성하여 그래프 배열에 push()합니다.

일론 머스크 글 본문에 AI라는 단어가 없어도, 원본 카테고리 텍스트 안에 'AI'라는 단어가 슬래시로 묶여 있었기 때문에 시스템이 이를 독립된 'AI' 테제 노드로 자동 인식하고 글을 바인딩하는 것입니다.

2. UI 대주제 명칭 치환 (정규식 및 단순 매치)
화면에 표시되는 AI 활용법(실전), 매크로·국제정세, 투자 철학·단상 같은 정제된 대주제 명칭은, 파싱 단계에서 post[2]의 원문 문자열 중 특정 핵심 키워드가 감지되면 화면 표시용 그룹 텍스트로 치환(Replace)되도록 조건문 처리가 되어 있습니다. (예: 투자단상이 포함되어 있으면 상위 레이블을 투자 철학·단상으로 그룹화)

3. 실시간 글 개수 카운팅 및 렌더링
각 노드가 동적으로 생성될 때, 해당 카테고리 분절 구조에 속한 포스트들의 고유 ID(post[0])를 자식 배열(posts 리스트)에 주입합니다.

최종적으로 이 배열의 길이(posts.length)를 계산하여 화면에 243, 104 같은 숫자를 실시간으로 뿌려주는 구조입니다.
