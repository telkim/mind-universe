## mind-univers 개발 지침

사용자와의 대화는 한국어로 한다.
코드, 변수명, 파일명은 영어를 사용한다.

## 프로젝트 개요

이 프로젝트는 마크다운 글을 디지털 가든 형태로 찾아보기 쉽게 보여주는 시스템이다.

## 기술 스택 (Environment)

* **Framework**: Quartz 4 (Hugo/Vue가 아닌 Node.js, TypeScript, Preact 기반 SSG)
* **Visualization**: Quartz 내장 Graph View (D3.js / Canvas 기반 렌더링 엔진)
* **CSS/Style**: Tailwind CSS 또는 Quartz 내장 정적 스타일시트 (카키/밤부 톤 다크모드 고정)
* **Storage**: 브라우저 LocalStorage (나중에 읽기, 스크랩 데이터 영속화용)


## 작업 전 필수 확인 문서

기능 개발이나 구조 변경 전 반드시 아래 파일명 패턴의 문서를 확인한다. (XX는 숫자)

docs/PROD_XX.md



## 개발 원칙

기존 명세와 충돌하는 구현은 하지 않는다.


"기능명세 전체를 한 번에 구현하지 않는다. 반드시 하나의 기능 단위로 쪼개고, 완료 가능한 상태까지 구현한다."

순서:

1. 요구사항 확인
2. 영향 범위 분석
3. 구현 계획 제시
4. 코드 변경
5. 테스트
6. 문서 업데이트 검토

---

기존 구조 존중

새로운 프레임워크나 라이브러리를 임의 추가하지 않는다.

기존 패턴을 우선 따른다.

---

문서 우선 개발

새로운 기능 추가 시:

1. 관련 PROD 문서 확인
2. 필요한 경우 PROD 문서 수정 제안
3. 코드 구현
4. 변경 내용 정리

---

6. Git 작업 규칙

작업 완료 후:

- 변경 파일 확인
- 변경 내용 요약
- 적절한 commit message 제안

예:

feat: add student attendance management
fix: resolve login session issue
docs: update PROD_03 database specification

사용자가 승인하면 commit 수행.

---

7. 코드 변경 규칙

코드 수정 전:

- 현재 파일 구조 확인
- 관련 코드 검색
- 영향 범위 설명

대규모 변경은 바로 실행하지 않고 계획을 먼저 제시한다.

---

8. AI 작업 방식

사용자가 기능 요청을 하면:

1. 요구사항 해석
2. 관련 PROD 문서 확인
3. 구현 계획 작성
4. 필요한 파일 목록 제시
5. 코드 변경

순서로 진행한다.

임의로 프로젝트 방향을 변경하지 않는다.

---

9. 답변 스타일

개발 관련 답변:

- 변경 이유 설명
- 수정 파일 목록 제공
- 실행 방법 제공
- 테스트 방법 제공

불필요하게 긴 문서를 생성하지 않는다.

구현 중심으로 진행한다.
